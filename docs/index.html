Start with semantic HTML and let legacy.css provide the administrative interface defaults.

Install

Add the package with your preferred package manager.

npm install legacy.css
pnpm add legacy.css
Use the CSS

Reference the built stylesheet from your HTML.

<link rel="stylesheet" href="./node_modules/legacy.css/dist/legacy.css">

Or import the source entry point from a bundler.

@import "legacy.css/src/legacy.css";
Add optional JavaScript

Tabs, modal dialogs, toast notifications, and drag/drop boards use the optional browser script.

<script src="./node_modules/legacy.css/dist/legacy.min.js" defer></script>
Write semantic HTML

Native elements such as forms, tables, buttons, fieldsets, and details receive usable defaults without extra classes.

<main>
  <h1>Supplier registration</h1>

  <form>
    <fieldset>
      <legend>Registration data</legend>

      <label for="name">Name</label>
      <input id="name" name="name" type="text">
    </fieldset>

    <button type="submit">Save</button>
  </form>
</main>
Use the MCP server

legacy.css includes a stdio MCP server so coding agents can inspect the library and quickly scaffold administrative pages with the right semantic elements and component classes.

Run the server from this repository:

pnpm run mcp

Configure an MCP client with a local checkout:

{
  "mcpServers": {
    "legacy-css": {
      "command": "pnpm",
      "args": ["run", "mcp"],
      "cwd": "/absolute/path/to/legacy.css"
    }
  }
}

After package installation, configure the executable directly:

{
  "mcpServers": {
    "legacy-css": {
      "command": "legacy-css-mcp"
    }
  }
}

Ask your agent to use the MCP server before writing markup:

Use the legacy-css MCP server to build a supplier registration page.
Include a toolbar, two fieldsets, status badges, a table of recent suppliers,
and an alert area for validation messages.

The server exposes resources for the README, source CSS entry point, built CSS, and every CSS module. It also provides tools to list modules, read a module, inspect design tokens, search selectors or properties, and return semantic HTML snippets for common page patterns.
